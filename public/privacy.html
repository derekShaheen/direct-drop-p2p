Privacy Policy

Effective date: 2026-02-12

DirectDrop is designed for private, browser-to-browser file sharing. This policy explains what we do and do not process.

What we do not collect or store

  • We do not upload, inspect, store, or retain your file contents on our server.
  • We do not create user accounts.
  • We do not store personal profiles, contact lists, or message history.
  • We do not persist transfer manifests (file names and sizes) as a long-term server record.

How transfers work

  • File bytes move directly between sender and receiver browsers over WebRTC data channels.
  • The server only helps peers connect by relaying signaling messages needed to establish the direct connection.
  • WebRTC transport is encrypted in transit.

What limited data is processed

  • Temporary connection state: short-lived room tokens and in-memory signaling/transfer state used to coordinate a live transfer session.
  • Public aggregate counters: total successful transfers, total files transferred, and total bytes transferred across all users.

These aggregate counters are not user-specific and are intended only as public service statistics.

Retention

  • Temporary in-memory transfer/session state expires automatically and is cleaned up on a short time window.
  • Only aggregate counters are persisted for display on the site.

Contact

If this deployment provides a support channel, use that channel for privacy questions related to this service.

This policy describes the DirectDrop application behavior in this repository and its default server configuration.
